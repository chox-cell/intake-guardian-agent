## 2025-05-18 - Prevent Host Header Injection and Open Redirects
**Vulnerability:** The application constructed absolute URLs using unvalidated `x-forwarded-proto` and `x-forwarded-host` headers and used these for internal redirects (`res.setHeader("location", ...)`). This could lead to Host Header Injection and Open Redirects if headers were manipulated.
**Learning:** Never trust HTTP headers like `x-forwarded-proto` and `x-forwarded-host` blindly. When constructing URLs from these headers, they must be validated against a strict allowlist. Additionally, internal application redirects should rely on relative paths rather than constructing absolute URLs based on user-supplied request headers.
**Prevention:** Validate `x-forwarded-proto` against allowed values (`http`, `https`) and validate `x-forwarded-host` using a strict domain regex (e.g., `/^[a-zA-Z0-9.-]+(:\d+)?$/`). Always use relative paths for internal `res.setHeader('location', ...)` or `res.redirect()` responses.
