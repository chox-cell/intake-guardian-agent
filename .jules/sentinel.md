## 2025-03-10 - Prevent CSV Formula Injection in ticketsToCsv
**Vulnerability:** The application exports user-controlled data to CSV format without properly sanitizing or escaping fields that begin with special characters (`=`, `+`, `-`, `@`). This leads to CSV Injection (Formula Injection), where opening the CSV in spreadsheet applications like Microsoft Excel can execute arbitrary formulas or commands.
**Learning:** Even if data is not considered active code in the application, it can become an execution vector when exported to external tools. Spreadsheets eagerly interpret any value starting with these special characters as a formula, which can be abused by an attacker to exfiltrate data or gain remote code execution on the client machine.
**Prevention:** Always sanitize data before generating CSVs. Any cell data that starts with a formula trigger character (`=`, `+`, `-`, `@`) must be escaped, usually by prepending a single quote (`'`), which instructs the spreadsheet program to treat the field as a plain text string instead of a formula.
