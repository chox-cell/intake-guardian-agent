## 2025-02-14 - [CRITICAL] Store Passwordless Auth Tokens as Hashes
**Vulnerability:** The application was storing plaintext passwordless authentication tokens (used for magic links) directly into `tokens.json`. If an attacker gained access to `tokens.json` (e.g., through directory traversal, LFI, or gaining server access), they could immediately use the tokens to impersonate users by generating verify URLs.
**Learning:** Tokens used for authentication, even temporary ones like magic links, represent a direct credential similar to a password. Storing them in plaintext violates the principle of failing securely because a single data breach compromises active sessions. We must treat them like passwords.
**Prevention:** Always store sensitive, high-value tokens (like magic link tokens, session tokens, or API keys) as cryptographic hashes (e.g., using SHA-256) at rest. Upon receiving the token from the user, calculate its hash and compare it against the stored hash using a constant-time comparison.
