## 2024-05-18 - [Fix Host Header Injection and Open Redirect]
**Vulnerability:** Host Header Injection in multiple `baseUrl` calculation functions throughout the codebase (`src/server.ts`, `src/ui/routes.ts`, `src/api/auth.ts`, `src/ui/start_route.ts`), where an unsanitized `Host` header was blindly trusted to construct absolute URLs, leading to potential Open Redirect and cache poisoning if this string was used in `Location` headers or exposed in HTML.
**Learning:** We constructed URLs based on `x-forwarded-host` or `host` headers without regex validation, returning user-controllable input into templates and redirects. Furthermore, relative paths should be used for internal redirects when possible (`res.setHeader("location", "/path")`).
**Prevention:** In functions computing URLs from requests (like `baseUrl`), always fall back explicitly to a known environment variable like `APP_BASE_URL` first. Before relying on `host` strings, rigorously validate them against a regex like `/^[a-zA-Z0-9.-]+(:\d+)?$/`. Use relative paths for `Location` headers on local internal redirects.
