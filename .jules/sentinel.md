## 2026-04-27 - [CSV Injection / Formula Injection in Export Utilities]
**Vulnerability:** Multiple CSV export utilities (e.g., `toCSV`, `ticketsToCsv`, `exportCsv`) across the application (`src/api/ui.ts`, `src/lib/ticket-store.ts`, etc.) did not sanitize user-supplied data before inserting it into CSV formats. If user input starts with `=`, `+`, `-`, or `@`, spreadsheet applications like Microsoft Excel or Google Sheets may interpret the cell contents as a formula.
**Learning:** Formula injection (also known as CSV injection) occurs when malicious payloads are inserted into CSV files. When opened in spreadsheet software, these payloads can execute commands, leak data, or launch other attacks on the user's local machine. This is a common risk when exporting data directly to CSV.
**Prevention:** To prevent CSV injection, prepend a single quote (`'`) to any cell value that starts with a formula character (`=`, `+`, `-`, `@`). This forces spreadsheet software to treat the contents as plain text rather than an executable formula. All CSV export functions should adopt a consistent escaping helper function.
