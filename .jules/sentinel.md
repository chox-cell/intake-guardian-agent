## 2025-02-05 - [CRITICAL] Fix Host Header Injection in base URL generation
**Vulnerability:** Found multiple instances where the application constructs base URLs using the raw `x-forwarded-host` or `host` headers without any validation (`baseUrl` functions in `src/server.ts` and `src/ui/routes.ts`, and `computeBaseUrl` in `src/api/auth.ts`). This allows an attacker to inject a malicious Host header to control the domain name in generated links, emails, and redirects, leading to potential phishing attacks or credential theft.
**Learning:** Functions that generate URLs or base URLs should never blindly trust request headers. While standard practices correctly identify the need to use `x-forwarded-host` or `host`, doing so without validation allows attackers to easily manipulate the generated URLs since HTTP headers are user-controlled.
**Prevention:** Always prioritize explicitly trusted configurations such as `process.env.APP_BASE_URL` when generating URLs. If falling back to request headers, rigorously validate the host string against a strict allowlist or regex (e.g., `^[a-zA-Z0-9.-]+(:\d+)?$`) before using it.
