## 2025-02-14 - Fix CSV Formula Injection Vulnerability
**Vulnerability:** Found missing escaping for strings starting with `=`, `+`, `-`, or `@` across multiple CSV export implementations (`src/api/ui.ts`, `src/lib/ticket-store.ts`, `src/lib/ticket_store.ts`, `src/lib/tickets_store.ts`). This allows malicious users to craft ticket data that executes spreadsheet formulas when the CSV is opened in Excel or similar applications.
**Learning:** CSV injections (or formula injections) occur when user input starting with formula characters is treated as executable code by spreadsheet software. This project has multiple discrete CSV export paths that all need the same mitigation logic, meaning fixing one doesn't secure the application entirely.
**Prevention:** Always prepend a single quote (`'`) to user-generated string values that start with formula characters (`=`, `+`, `-`, `@`) during CSV serialization. Consolidate CSV generation logic where possible to avoid scattered instances of vulnerable implementations.
