## 2025-01-20 - [Fix Host Header Injection and Open Redirect]
**Vulnerability:** The application was vulnerable to Host Header Injection and Open Redirect. The `baseUrl` function in `src/server.ts` constructed a base URL using the `host` header (or `x-forwarded-host`) without validating it, allowing an attacker to inject an arbitrary host. This injected host was then used in a 303 redirect (`res.setHeader("location", ...)`), leading to an Open Redirect vulnerability.
**Learning:** Request headers such as `host` and `x-forwarded-host` can be easily manipulated by an attacker. Relying on them directly to construct URLs for redirects or resource loading without validation creates a significant security risk, especially in environments without strict routing rules.
**Prevention:** To prevent this, always prioritize a trusted configuration value (like `process.env.APP_BASE_URL`). If fallback to request headers is necessary, rigorously validate the host string against a strict allowlist or a strict regex (e.g., `/^[a-zA-Z0-9.-]+(:\d+)?$/`). Furthermore, use relative paths for internal redirects whenever possible to completely avoid Open Redirect issues regardless of the host.
