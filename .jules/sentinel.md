## 2025-03-14 - Fix CSV Formula Injection in Ticket Export
**Vulnerability:** The application was vulnerable to CSV injection because the `ticketsToCsv` function did not properly sanitize fields that started with execution characters like `=`, `+`, `-`, or `@`.
**Learning:** Even though the function correctly escaped double quotes and handled line breaks by wrapping values in double quotes, this isn't enough to prevent spreadsheet applications (like Excel) from executing a formula injected into the CSV. If a user maliciously names their ticket starting with `=`, it could run arbitrary macros upon opening.
**Prevention:** Added logic inside the internal `esc` function of `ticketsToCsv` to explicitly test if a string starts with any of the dangerous characters (`^[=+\-@]`) and if so, prepend a single quote (`'`). This prevents formula execution in common spreadsheet software. Always escape CSV payloads against formula injections, not just structure-breaking characters.
