## 2025-05-24 - [CRITICAL] Fix Path Traversal in File Storage Paths
**Vulnerability:** The application is vulnerable to path traversal because `tenantId` is passed directly to `path.join` and `path.resolve` when constructing file paths for `tickets.json` in `src/lib/ticket-store.ts`, `src/lib/tickets_disk.ts`, `src/lib/ticket_store.ts`, `src/lib/tickets_store.ts`, and `src/lib/tickets_pipeline.ts`. An attacker could provide a `tenantId` like `../../../etc/passwd` to read or write sensitive files outside the intended data directory.
**Learning:** When using user input to construct file paths, the input must always be explicitly validated against a strict allowlist (e.g., regex `^[a-zA-Z0-9_-]+$`) to ensure it only contains safe characters and does not include traversal sequences like `..` or `/`. `path.join` and `path.resolve` are not sufficient protection on their own.
**Prevention:** Always validate and sanitize user inputs that are used in file system operations. Enforce strict character sets (e.g., alphanumeric, underscores, and hyphens) for identifiers like `tenantId` before they are used in any path manipulation.
